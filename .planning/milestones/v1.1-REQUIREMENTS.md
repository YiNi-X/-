# Milestone v1.1 Requirements Archive: Offline Showcase Expansion

**Archived:** 2026-03-30
**Outcome:** 31 / 31 v1.1 requirements validated

## v1 Requirements

### Showcase Foundation

- [x] **BASE-01**: Demo viewer can navigate between overview, flow prediction, trajectory repair, trajectory clustering, collaborative decision, and evaluation from one coherent website shell
- [x] **BASE-02**: Website loads module data from committed static artifacts derived from archived local research outputs instead of requiring a live backend or online inference
- [x] **BASE-03**: Demo viewer can switch scenarios or time slices without broken module state or missing data contracts
- [x] **BASE-04**: Website lazy-loads module or scenario artifacts so new datasets do not force one monolithic initial payload
- [x] **BASE-05**: Team can regenerate website-facing showcase artifacts from `代码依据` and existing cleaned outputs through one documented offline pipeline
- [x] **BASE-06**: Team can run one repeatable verification flow that checks buildability and critical module data contracts before demoing the site

### Flow Prediction

- [x] **FLOW-01**: Demo viewer can switch among STGCN, LSTM, and BiLSTM prediction results for the same archived scenario
- [x] **FLOW-02**: Demo viewer can switch forecast horizon between 1h, 2h, and 3h
- [x] **FLOW-03**: Demo viewer can inspect key prediction metrics for the selected model and horizon, including MAE, RMSE, and R-squared
- [x] **FLOW-04**: Demo viewer can inspect hotspot grid ranking and total flow trends during historical playback

### Trajectory Repair

- [x] **REPR-01**: Demo viewer can choose from curated sample trajectories that contain missing or damaged segments
- [x] **REPR-02**: Demo viewer can compare original ground truth, missing trajectory, and repaired trajectory on the same visual surface
- [x] **REPR-03**: Demo viewer can switch among ATT-BILSTM, BiLSTM, LSTM, and at least one classical baseline for the selected repair sample
- [x] **REPR-04**: Demo viewer can inspect longitude and latitude error charts for the selected repair result
- [x] **REPR-05**: Demo viewer can inspect a repair metric summary that includes MAE, RMSE, DTW, R-squared, and Hausdorff distance

### Trajectory Clustering

- [x] **CLUS-01**: Demo viewer can switch between original trajectories, segmented trajectories, compressed trajectories, and the shipped corridor extraction views where truthful artifacts exist
- [x] **CLUS-02**: Demo viewer can inspect cluster count statistics, corridor dominance, and honest pre-reclustering noise-pool proportions for the shipped clustering run
- [x] **CLUS-03**: Demo viewer can compare first-pass clustering with noise re-clustering results once a readable distance artifact is restored; until then the site must expose an honest deferred CLUS-03 boundary instead of mocked visuals
- [x] **CLUS-04**: Demo viewer can inspect main corridor extraction results and link them back to dashboard or map entities used elsewhere in the site

`CLUS-03` outcome note: `v1.1` validated the deferred-boundary half of the requirement by shipping `clustering-noise-fallback.json`, a zero-byte artifact status report for `normalized_distances(60,90,0.03).pkl`, and consistent deferred messaging across clustering, evaluation, overview, and forward-looking. A true noise re-clustering comparison remains future scope.

### Collaborative Decision

- [x] **DECI-01**: Demo viewer can see the current focus route and focus grid for a selected frame or scenario
- [x] **DECI-02**: Demo viewer can inspect strategy suggestions associated with the selected frame or scenario
- [x] **DECI-03**: Demo viewer can toggle between before-strategy and after-strategy states and compare benefit changes
- [x] **DECI-04**: Demo viewer can read explanation copy showing which forecast or clustering evidence drove the suggested strategy

### Evaluation Center

- [x] **EVAL-01**: Demo viewer can inspect unified metric cards or tables across prediction and repair tasks using the metrics appropriate to each task
- [x] **EVAL-02**: Demo viewer can rank flow prediction models by selected horizon and metric
- [x] **EVAL-03**: Demo viewer can rank trajectory repair models by selected sample and metric
- [x] **EVAL-04**: Demo viewer can inspect parameter optimization history and parameter importance for supported offline models
- [x] **EVAL-05**: Team can trace displayed evaluation numbers back to committed offline result artifacts

### Project Overview

- [x] **OVER-01**: Overview page explains the end-to-end business loop from archived AIS playback through clustering or repair, prediction, and collaborative decision
- [x] **OVER-02**: Overview page surfaces data scale, scenario entry points, and direct links into each algorithm module instead of low-value system logs
- [x] **OVER-03**: Overview page clearly frames the product as archived playback plus offline inference rather than live AIS

## Carry-Forward Candidates

These were never part of the shipped `v1.1` commitment, but they remain the clearest next-milestone candidates:

- `LIVE-01`: Website can ingest or query realtime AIS data from a dependable external source
- `LIVE-02`: Website can execute prediction or repair inference online for newly arriving data
- `LIVE-03`: Website can manage scenarios, runs, and result history through a backend service instead of committed static artifacts
- `LIVE-04`: User can upload custom trajectory files and receive on-demand analysis
