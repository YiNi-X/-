# Milestone v1.1: Offline Showcase Expansion

**Status:** SHIPPED 2026-03-30
**Phases:** 6-12
**Total Plans:** 22 roadmap plans

## Overview

`v1.1` turned the earlier offline dashboard into a modular showcase website backed by committed archived-data bundles. The shipped surface now covers overview, flow prediction, trajectory repair, trajectory clustering, evaluation, and forward-looking decision support through one shared shell, while keeping the site's truth boundary explicit: archived AIS playback, offline-generated evidence, no live optimizer, and no fake noise re-clustering.

## Milestone Summary

**Key Decisions**

- Keep the product as an archived-data showcase rather than pretend live AIS infrastructure exists.
- Use module manifests, bundles, and lazy loading as the long-term scaling model for the frontend.
- Let corridor dominance become a site-wide narrative spine rather than a clustering-only footnote.
- Ship an honest `CLUS-03` fallback contract while `normalized_distances(60,90,0.03).pkl` remains zero-byte.
- Treat collaborative decision as a rule-driven evidence layer over shipped forecast and clustering context, not a live optimizer.

**Issues Resolved**

- Replaced one dense demo page with a coherent multi-module product shell.
- Converted notebook-heavy forecast, repair, clustering, evaluation, and overview evidence into committed website-facing bundles.
- Added site-wide traceability between module storylines, especially around evaluation authority and corridor dominance.
- Closed forward-looking as a ready module instead of leaving it as a placeholder or partial route.

**Issues Deferred**

- Real noise re-clustering comparison remains deferred until `normalized_distances(60,90,0.03).pkl` is restored as a readable non-zero artifact.
- Node-level forecast backfill and paper-style animation remain optional enhancement work, not shipped milestone scope.
- Live AIS ingestion, online inference, and backend scenario management remain future-milestone questions.

**Technical Debt Incurred**

- Later phase verification is recorded in summary closeout text more often than in standalone `*-VERIFICATION.md` files.
- Some later executed waves do not have matching standalone `*-PLAN.md` files, so archival is accurate but not perfectly uniform.
- Git history still needs a clean milestone commit and tag if the repo owner wants the archival state reflected in commit history.
