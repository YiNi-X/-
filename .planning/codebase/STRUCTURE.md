# Codebase Structure

**Analysis Date:** 2026-03-23

## Directory Layout

```
[project-root]/
- demo-web/                React + data project with UI, assets, scripts, and public data
- .planning/               GSD mapping output; only `.planning/codebase/` is mutated here
- .vercel/                 Vercel metadata kept by deployment
- netlify.toml             Netlify/deployment config
- vercel.json              Vercel overrides for routing/service workers
- 代码依据/                Supporting reference docs (read-only for mapping)
- 相关图片提取/            Reference imagery/assets extracted for the project
- generate_ai_collab_pdf.py Standalone helper script outside `demo-web`
```

## Directory Purposes

**demo-web/:**
- Purpose: housing the Vite/React dashboard, RouteEditor entry, static assets, and supporting scripts.
- Contains: `src/` (code), `public/` (HTML/JSON assets), `scripts/` (Python pipelines), `analysis/` (exported CSV/JSON), `dist/` (build output), configs (`package.json`, `vite.config.ts`, `tsconfig*.json`, `netlify.toml`, `vercel.json`).
- Key files: `demo-web/package.json`, `demo-web/vite.config.ts`, `demo-web/tsconfig.app.json`.

**demo-web/src/:**
- Purpose: compile-time TypeScript entry points, helpers, and styles that render the dashboard (`App.tsx`) and editor (`RouteEditor.tsx`).
- Contains: React components, data/geometry helpers (`datasetCatalog.ts`, `sharedContracts.ts`, `scenarioPacks.ts`, `mainCorridorSelection.ts`), global styles (`App.css`, `route-editor.css`, `index.css`), and `assets/`.
- Key files: `demo-web/src/App.tsx`, `demo-web/src/RouteEditor.tsx`, `demo-web/src/index.css`.

**demo-web/src/assets/:**
- Purpose: small static imagery imported directly into components (e.g., hero images, SVG logos).
- Contains: `hero.png`, `react.svg`, `vite.svg`.
- Key files: `demo-web/src/assets/hero.png`.

**demo-web/public/:**
- Purpose: the Vite public directory that exposes `index.html`, `route-editor.html`, SVG icons, and the `data/` folder at the root.
- Contains: HTML entry points, `favicon.svg`, `icons.svg`, `static-port-map.jpg`, and `data/`.
- Key files: `demo-web/index.html`, `demo-web/route-editor.html`.

**demo-web/public/data/:**
- Purpose: runtime JSON datasets consumed by both entries (AIS playback, forecasts, geometry, scenario catalogs, and corridor exports).
- Contains: `ais-playback.json`, `flow-forecast.json`, `dataset-catalog.json`, `shared-geometry.json`, `model-config.json`, `main-corridor-tracks.json`, plus CSV/JSON references generated by the Python scripts.
- Key files: `demo-web/public/data/ais-playback.json`, `demo-web/public/data/main-corridor-tracks.json`.

**demo-web/scripts/:**
- Purpose: offline tooling that generates the datasets and corridor exports consumed at runtime.
- Contains: `generate_first_version_data.py`, `extract_main_corridors_from_clustered_ais.py`, `stgcn_runtime.py`.
- Key files: `demo-web/scripts/generate_first_version_data.py`.

**demo-web/analysis/:**
- Purpose: derived analytics output (CSV/JSON) produced by the scripts for review or re-ingestion.
- Contains: `clustered_ais_vs_website_dataset_summary.json`, `senior_main_corridors_summary.json`, `*.csv`.
- Key files: `demo-web/analysis/senior_main_corridors_summary.json`.

**demo-web/dist/:**
- Purpose: build output from `npm run build`; contains pre-bundled `route-editor.html` plus hashed assets.
- Contains: Vite-generated assets that mirror `src/` output.
- Key files: `demo-web/dist/route-editor.html`.

**demo-web/node_modules/:**
- Purpose: dependency install directory created by `npm install`.
- Contains: all packages required by Vite, React, TypeScript, and ESLint.
- Key files: (managed by npm; do not edit manually).

**.planning/codebase/:**
- Purpose: stores GSD codebase mapping documents like `CONVENTIONS.md`, `TESTING.md`, and the new `ARCHITECTURE.md`/`STRUCTURE.md`.
- Contains: mapping artifacts only.

**.vercel/:**
- Purpose: deployment metadata (build cache, project settings) generated by Vercel.
- Contains: runtime config files Vercel regenerates.

**代码依据/ & 相关图片提取/:**
- Purpose: reference material imported from the broader project (do not edit while mapping).

## Key File Locations

**Entry Points:**
- `demo-web/index.html`: loads `/src/main.tsx` (dashboard entry).
- `demo-web/route-editor.html`: boots the editor and redirects to `demo-web/dist/route-editor.html` if available.
- `demo-web/src/main.tsx`: renders `<App />`.
- `demo-web/src/route-editor.tsx`: renders `<RouteEditor />`.

**Configuration:**
- `demo-web/package.json`: scripts (`dev`, `editor`, `build`, `generate:data`) and dependency list.
- `demo-web/vite.config.ts`: registers `main` and `routeEditor` inputs, sets base URL `./`.
- `demo-web/tsconfig.json`, `demo-web/tsconfig.app.json`, `demo-web/tsconfig.node.json`: bundler-aware compiler settings.
- `demo-web/eslint.config.js`: ESLint flat config referencing `@eslint/js`, TypeScript, and React hooks.
- `demo-web/netlify.toml`, `demo-web/vercel.json`: static deployment overrides.

**Core Logic:**
- `demo-web/src/App.tsx`: entire dashboard, including playback controls, map layers, strategy/feed/benefit panels.
- `demo-web/src/RouteEditor.tsx`: corridor editor stage, inspector, and export controls.
- `demo-web/src/datasetCatalog.ts`: dataset catalog loading, sanitization, persistence, and URL resolution.
- `demo-web/src/sharedContracts.ts`: typed domain models consumed across the app.
- `demo-web/src/scenarioPacks.ts`: map tag/focus data and benchmark definitions.

**Testing:**
- Not present; add manual QA or future tests near the corresponding `src/` component when needed.

## Naming Conventions

**Files:**
- PascalCase for React components (`demo-web/src/App.tsx`, `demo-web/src/RouteEditor.tsx`).
- camelCase helpers for data modules (`demo-web/src/datasetCatalog.ts`, `demo-web/src/mainCorridorSelection.ts`, `demo-web/src/scenarioPacks.ts`).
- Kebab-case CSS for layout files (`demo-web/src/App.css`, `demo-web/src/route-editor.css`).
- Hyphenated lowercase for static JSON (`demo-web/public/data/ais-playback.json`, `demo-web/public/data/flow-forecast.json`).

**Directories:**
- Lowercase directories describe purpose: `src`, `public`, `scripts`, `analysis`, `dist`. PascalCase directories are not used.

## Where to Add New Code

**New Feature:**
- Primary code: add a new `tsx`/`ts` file under `demo-web/src/`, hook it into `demo-web/src/App.tsx` or a dedicated submodule, and include any styles in `demo-web/src/App.css` or a new stylesheet imported by the component.
- Tests: not defined yet; place future test files adjacent to the component created under `demo-web/src/`.

**New Component/Module:**
- Implementation: `demo-web/src/<feature>/` (create a folder if needed), import it from `demo-web/src/App.tsx` or `demo-web/src/RouteEditor.tsx`, and keep data/contract definitions in `demo-web/src/sharedContracts.ts` if shared.

**Utilities:**
- Shared helpers: extend `demo-web/src/sharedContracts.ts`, `demo-web/src/datasetCatalog.ts`, or create a new `demo-web/src/<utility>.ts` and export strongly typed helpers used by both entries.

## Special Directories

**`demo-web/dist/`:**
- Purpose: Vite build artifacts served in production; used by the redirect within `demo-web/route-editor.html`.
- Generated: Yes (recreated by `npm run build`).
- Committed: Yes (checked in to keep distribution files in sync).

**`demo-web/analysis/`:**
- Purpose: stores CSV/JSON summaries produced by `demo-web/scripts/extract_main_corridors_from_clustered_ais.py`.
- Generated: Yes (update via the Python scripts).
- Committed: Yes (the artifacts accompany the mapping work but are not part of the interactive UI).

**`demo-web/node_modules/`:**
- Purpose: dependency install cache for Vite, React, ESLint, and TypeScript.
- Generated: Yes (npm install).
- Committed: No (should be ignored, but it is present for local builds; do not edit).
