# External Integrations

**Analysis Date:** 2026-03-23

## APIs & External Services

**Static dataset artifacts:** 
- `demo-web/public/data/dataset-catalog.json` – consumed by `demo-web/src/datasetCatalog.ts` (`loadDatasetCatalog`) to determine available dataset IDs, with query parameter `dataset` and fallback localStorage key `route-motion.dataset-id`.
  - SDK/Client: native `fetch` inside `demo-web/src/datasetCatalog.ts`.
  - Auth: Not applicable.
- `demo-web/public/data/ais-playback.json` & `demo-web/public/data/flow-forecast.json` – fetched by `demo-web/src/App.tsx` via `loadPublicJson` (from `demo-web/src/sharedContracts.ts`) to populate playback frames, forecast timeline, narratives, and summary cards.
- `demo-web/public/data/shared-geometry.json` – read by `demo-web/src/App.tsx` (through `loadPublicJson` / `resolveRuntimeResource`) for study bounds, route geometry, and hotspot metadata that drive map tags and angle calculations.
- `demo-web/public/data/main-corridor-tracks.json` – loaded in `demo-web/src/RouteEditor.tsx` (`loadPublicJson`) for corridor track editing, label positioning, and handle dragging.
- `demo-web/public/data/model-config.json` – generated by `demo-web/scripts/generate_first_version_data.py` and surfaced indirectly through `demo-web/public/data/flow-forecast.json` metadata.
- `demo-web/public/data/demo-live-ais-*.csv` – generated reference AIS CSV files served alongside the JSON assets for reproducibility.

## Data Storage

**Databases:**
- None; playback, forecast, catalog, and geometry data live solely as files under `demo-web/public/data/` (e.g., `demo-web/public/data/ais-playback.json`, `demo-web/public/data/flow-forecast.json`, `demo-web/public/data/dataset-catalog.json`, `demo-web/public/data/model-config.json`, `demo-web/public/data/shared-geometry.json`, `demo-web/public/data/main-corridor-tracks.json`, and `demo-web/public/data/demo-live-ais-*.csv`).

**File Storage:**
- Local repository assets include `demo-web/public/static-port-map.jpg` plus analysis dumps under `demo-web/analysis/` (such as `demo-web/analysis/senior_main_corridors_summary.json`, `demo-web/analysis/senior_main_corridors_cleaned_points.csv`, `demo-web/analysis/senior_main_corridors_track_stats.csv`, and `demo-web/analysis/clustered_ais_vs_website_dataset_summary.json`). These artifacts are generated by `demo-web/scripts/generate_first_version_data.py` and `demo-web/scripts/extract_main_corridors_from_clustered_ais.py`, which read raw inputs from `代码依据/轨迹聚类/cleaned_ais.CSV`, `代码依据/轨迹聚类/compressed_segments(60,90,0.03).pkl`, `代码依据/流量预测/grid_mmsi_count（流量数据）.csv`, and `代码依据/流量预测/save/model_0.pt`.

**Caching:**
- No explicit cache layer; the browser and hosting CDN cache the static JSON/CSV assets, while dataset choice is persisted in `window.localStorage` (`route-motion.dataset-id`) and the `dataset` query parameter via `demo-web/src/datasetCatalog.ts`.

## Authentication & Identity

**Auth Provider:**
- None; the SPA only requests static files from the same origin.

## Monitoring & Observability

**Error Tracking:**
- None configured.

**Logs:**
- None beyond front-end console logs and standard output from `demo-web/scripts/*.py`.

## CI/CD & Deployment

**Hosting:**
- Vercel uses `vercel.json` (`installCommand`: `cd demo-web && npm install`, `buildCommand`: `cd demo-web && npm run build`, `outputDirectory`: `demo-web/dist/`).
- Netlify uses `netlify.toml` (base `demo-web`, `command` `npm run build`, `publish` `dist`).

**CI Pipeline:**
- Deployments rely on `npm run build` from `demo-web/package.json` (runs `tsc -b && vite build`), so no separate CI tooling is required.

## Environment Configuration

**Required env vars:**
- None; configuration is baked into the bundled JSON assets and `route-editor.html`.

**Secrets location:**
- Not applicable; no secrets are stored.

**Dataset selection state:**
- `demo-web/src/datasetCatalog.ts` reads the `dataset` URL parameter, persists selections with `route-motion.dataset-id`, and rewrites the browser history (`window.history.replaceState`) to keep the query string in sync.

## Webhooks & Callbacks

**Incoming:**
- None; `index.html` and `route-editor.html` are static assets with no webhook endpoints.

**Outgoing:**
- None.
